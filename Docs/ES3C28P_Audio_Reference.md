# ES3C28P Audio Reference

Board: LCDWIKI ES3C28P — ESP32-S3, 2.8" IPS (ILI9341V), ES8311 codec, FM8002E 2W BTL amp.

---

## Audio chain

```
SD card → ESP32-S3 (I2S master) → ES8311 DAC (I2C 0x18) → FM8002E amp (BTL) → speaker
```

- ESP32-S3 drives MCLK, BCLK, LRCK, and sends PCM data to the ES8311 over I2S.
- ES8311 converts digital audio to analog and drives the FM8002E.
- FM8002E is a class-D BTL amplifier with an active-LOW shutdown pin.

---

## Pin assignments

| Signal       | GPIO | Notes |
|--------------|------|-------|
| I2S MCLK     | IO4  | Master clock, generated by ESP32 |
| I2S BCLK     | IO5  | Bit clock |
| I2S LRCK     | IO7  | Word select / LR clock |
| **I2S DOUT** | **IO8** | **ESP32 → ES8311 SDIN (DAC input). This is labelled I2S_DI on the board silkscreen.** |
| I2S DIN      | IO6  | ES8311 SDOUT → ESP32 (ADC output, mic recording only). Labelled I2S_DO on silkscreen. |
| I2C SDA      | IO16 | Shared with capacitive touch controller |
| I2C SCL      | IO15 | Shared with capacitive touch controller |
| SD CLK       | IO38 | |
| SD CMD       | IO40 | |
| SD D0        | IO39 | |
| SD D1        | IO41 | |
| SD D2        | IO48 | |
| SD D3        | IO47 | |
| Amp enable   | IO1  | FM8002E SHUTDOWN — active LOW (LOW = amp on, HIGH = off) |
| Battery ADC  | IO9  | 2:1 resistor divider, reads half of battery voltage (0–3.3 V range → 0–4.2 V actual) |

### Critical gotcha — I2S data pins are named from the codec's perspective

The silkscreen labels `I2S_DO` and `I2S_DI` describe the ES8311's own pins:

- `I2S_DO` (IO6) = ES8311 **SDOUT** — the codec *outputs* ADC data here (mic → ESP32). The ESP32 reads this.
- `I2S_DI` (IO8) = ES8311 **SDIN** — the codec *receives* DAC data here (ESP32 → speaker). The ESP32 writes this.

**For playback, `PIN_I2S_DOUT` must be IO8, not IO6.** Using IO6 causes complete silence because it is an output from the codec and cannot receive data.

## Library

[schreibfaul1/ESP32-audioI2S](https://github.com/schreibfaul1/ESP32-audioI2S) handles WAV/MP3/AAC decoding and I2S output. It does **not** initialize the ES8311; that must be done separately via direct I2C register writes.

`platformio.ini` dependency:
```ini
lib_deps =
    https://github.com/schreibfaul1/ESP32-audioI2S.git#2.0.6
```

Volume is set with `audio.setVolume(n)` where n is 0–21 (21 = max). A comfortable listening level for this board is around 8–10.

## ES8311 initialization

The ES8311 is in **slave mode** — ESP32 drives all clocks. I2C address is `0x18`.

MCLK must be running before the ES8311 is initialized, so call `audio.setPinout()` and `i2s_set_sample_rates()` first, wait ~20 ms, then write the registers.

### Clock setup for 44100 Hz

MCLK = 44100 × 256 = 11.2896 MHz (generated by ESP32 APLL).
BCLK = MCLK / 8 = 1.4112 MHz.
LRCK period = 256 BCLK cycles → 44100 Hz.

### Register sequence

| Reg  | Value | Purpose |
|------|-------|---------|
| 0x00 | 0x1F  | Soft reset all subsystems |
| 0x00 | 0x80  | Release reset; CSM on, slave mode |
| 0x01 | 0x3F  | All clocks enabled, MCLK from pin |
| 0x02 | 0x00  | MCLK pre-divider = 1, pre-multiplier = ×1 |
| 0x03 | 0x10  | ADC clock OSR |
| 0x04 | 0x10  | DAC clock OSR |
| 0x05 | 0x00  | ADC/DAC divider = 1 |
| 0x06 | 0x03  | BCLK divider = 8 |
| 0x07 | 0x00  | LRCK high byte = 0 |
| 0x08 | 0xFF  | LRCK low byte = 255 → period of 256 |
| 0x09 | 0x0C  | SDPIN (DAC input): standard I2S, 16-bit. Bits [5:4]=00 = standard I2S (NOT left-justified) |
| 0x0A | 0x0C  | SDPOUT (ADC output): standard I2S, 16-bit |
| 0x0B | 0x00  | |
| 0x0C | 0x00  | |
| 0x0D | 0x01  | Power up VMID reference |
| 0x0E | 0x02  | Power up reference voltage |
| 0x0F | 0x44  | HP driver bias current |
| 0x10 | 0x1F  | Analog power |
| 0x11 | 0x7F  | DAC/HP analog power |
| 0x12 | 0x00  | Output routing, DAC enabled |
| 0x13 | 0x10  | Headphone output |
| 0x14 | 0x1A  | HP mixer gain |
| 0x15 | 0x40  | ADC HPF on |
| 0x16 | 0x24  | ADC config |
| 0x17 | 0xBF  | ADC volume |
| 0x1B | 0x0A  | |
| 0x1C | 0x6A  | |
| 0x31 | 0x00  | DAC digital: L+R unmuted |
| 0x32 | 0xFF  | DAC digital volume L = 0 dB (max) |
| 0x33 | 0xFF  | DAC digital volume R = 0 dB (max) |
| 0x37 | 0x08  | DAC output enable, bypass equalizer (normal operating state) |
| 0x45 | 0x00  | GP control |
| 0x09 | 0x0C  | SDPIN: unmuted (bit 6 = 0), 16-bit I2S |
| 0x0A | 0x4C  | SDPOUT: muted (bit 6 = 1) — ADC not used for playback |

### Format gotcha

REG09 bits [5:4] select the serial format:
- `00` = standard I2S ← correct
- `01` = left-justified ← wrong; causes complete silence

Always write `0x0C` (WL=16-bit, FMT=standard I2S), never `0x10` or `0x1C`.

## SD card

4-bit SDMMC. Mount point `/sdcard`. Falls back to 1-bit if 4-bit fails.

```cpp
SD_MMC.setPins(PIN_SD_CLK, PIN_SD_CMD, PIN_SD_D0,
               PIN_SD_D1,  PIN_SD_D2,  PIN_SD_D3);
SD_MMC.begin("/sdcard", false);  // false = 4-bit mode
```

WAV files should be 44100 Hz, 16-bit, mono or stereo. Place them under `/sounds/` on the SD card root.

---

## Battery monitoring

IO9 sits on a 2:1 resistor divider. ADC reads half the battery voltage.

```
actual_voltage = adc_reading * (3.3 / 4095) * 2
```

LiPo range: 3.0 V (0%) to 4.2 V (100%).

## Initialization order (important)

1. `Wire.begin()` — I2C for ES8311
2. `audio.setPinout()` + `i2s_set_sample_rates()` — starts MCLK
3. `delay(20)` — let MCLK stabilize
4. `initES8311()` — codec registers via I2C
5. `digitalWrite(PIN_PA_EN, LOW)` — enable FM8002E amp
6. `audio.setVolume(n)` — software volume
7. `SD_MMC.begin()` — mount SD card
8. `audio.connecttoFS()` — begin playback
